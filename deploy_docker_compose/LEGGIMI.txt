==========================================================
  ISTRUZIONI DEPLOY - Assistente Condominio (WhatsApp + Voce)
==========================================================

Serve solo Docker installato sul server. NON serve git, NON serve
scaricare il codice: l'immagine e' gia' pronta.

Tieni questi 3 file nella STESSA cartella:
  - condominio-app.tar.gz   (immagine Docker dell'app)
  - docker-compose.yml
  - .env                    (chiavi/segreti: OpenAI, WhatsApp, Gmail, ngrok)

----------------------------------------------------------
PASSI
----------------------------------------------------------

1) Carica l'immagine in Docker (la prima volta e ad ogni nuova versione):

   docker load -i condominio-app.tar.gz

2) Avvia tutto:

   docker compose up -d

3) Trova l'URL pubblico ngrok (da incollare su Meta/WhatsApp e su Twilio):

   apri nel browser ->  http://IP_DEL_SERVER:4040
   (oppure)            docker compose logs ngrok

   Endpoint da configurare:
     WhatsApp (Meta -> webhook)            ->  https://<URL-NGROK>/webhook
     Voce (Twilio numero -> "A call comes in") -> https://<URL-NGROK>/voice/incoming

   La dashboard risponde anche direttamente su:  http://IP_DEL_SERVER:9999
   (gli stessi endpoint, con tasto "copia", sono anche in
    Dashboard -> icona ingranaggio -> Impostazioni tecniche)

----------------------------------------------------------
COMANDI UTILI
----------------------------------------------------------

  docker compose logs -f      # log in tempo reale
  docker compose down         # ferma tutto
  docker compose up -d        # riavvia

----------------------------------------------------------
NOTE
----------------------------------------------------------

- Persistenza: DB e documenti caricati stanno nella cartella ./data
  (condominio.db + documenti/). Al primo avvio il DB viene creato e
  popolato con dati demo (un condominio con alcuni inquilini e un ticket).

- Configurazione: le chiavi stanno nel .env e si possono modificare a caldo
  dalla dashboard (icona ingranaggio): il salvataggio riscrive il .env.
  Alcune (mittente email, modello/effort del risponditore) valgono subito;
  per le altre conviene riavviare il container.

- ngrok: nel .env serve un NGROK_AUTHTOKEN (account gratuito su ngrok.com).
  Per un URL fisso (consigliato, cosi' non riconfiguri WhatsApp/Twilio ad
  ogni riavvio) imposta NGROK_DOMAIN nel .env e segui il commento dentro
  docker-compose.yml.

- Ricostruire l'immagine dopo modifiche al codice (dalla cartella del
  progetto, dove c'e' il Dockerfile):
     docker build -t risponditore-condominio-app:latest .
     docker save risponditore-condominio-app:latest | gzip > deploy_docker_compose/condominio-app.tar.gz

- Questi file contengono chiavi private: NON condividerli pubblicamente.
